# 顾子明文章精读模板

> 用途: ZST 知识库增量入库的标准流程。每篇文章按此模板处理。

## 0. 准备

```
输入: 文章 HTML（来自 WeRSS 抓取）
工具:
  - 必用: 子 Agent + 多搜索工具（BrightData MCP、AnySearch、webctl）
  - 备选: 裸 WebSearch（仅对中文结果有效，国际新闻用子 Agent）
参考:
  - data-knowledge-bases/zst-kb/data/overview.json
  - data-knowledge-bases/zst-kb/data/articles/doc_zst_r000.json （示例）
  - data-knowledge-bases/zst-kb/data/claims.json （核验格式）
```

## 1. 清洗

```bash
# 从 WeRSS 读全文
sqlite3 F:/codex/output/we-mp-rss/data/db.db \
  "SELECT content FROM articles WHERE id = '<article_id>'" | \
  sed 's/<[^>]*>//g' | \
  sed 's/&nbsp;/ /g; s/&amp;/\&/g; s/&lt;/\</g; s/&gt;/\>/g; s/&quot;/"/g'
```

输出: 纯文本。

## 2. 初步判断 fq 信号

| 标志 | fq | 进入预测追踪 |
|------|-----|--------------|
| 含"我预测" / "未来 X 个月内" / 明确时点声明 | **S** | ✅ |
| 时事分析+作者框架但无预测 | A | 仅入库 |
| 纯历史回顾/事实陈述 | A | 仅入库 |

**经验数据**: 95.6% 文章 fq=A，4.4% 文章 fq=S。S 标记触发 claim 提取。

## 3. core_claim 提取（仅 fq=S）

模板（填三个空）:

```
<人/事/局>在<触发事件>后<动作>，作者判断<核心结论>。
```

检验:
- 30 字内能讲完
- 看完文章和看完这句话信息量相同
- 不带"我认为"——是结论不是意见

### il 标记

| 类型 | 标志 | 例子 |
|------|------|------|
| `author_model` | 文章用一套框架/模型解释 | "TACO 三方默契"、"王车易位"、"斩杀线" |
| `author_claim` | 文章有明确断言 | "公权力为衙内背书" |
| `source_text` | 引用具体制度/数据 | "老同志排序分组规则" |
| `''` | 难以归入上述三类 | — |

经验: 99% 文章 il=空。仅在能清晰识别框架/断言/引述时标注。

## 4. 预测识别（fq=S 文章必做）

模板:

```json
{
  "claim": "<预测文本>",
  "window_end": "<YYYY-MM-DD 或事件>",
  "type": "explicit|implicit"
}
```

### 显性预测（explicit）

- "我预测 X 会在 Y 时点发生"
- "未来 X 个月..."
- "即将..."
- 明确的时间锚点 + 状态改变

### 隐性预测（implicit）

- "学生们来不及 X，所以难以挑战 Y"
- "不出 X 个月，Y 必然..."
- 隐含的时序/概率判断

每篇文章可能有多条预测，每条独立写入 claims.json。

## 5. 事实核验

### 必查清单

| 类别 | 用什么查 |
|------|---------|
| 政治事件（选举、辞职、访问） | 子 Agent + BrightData + Euronews/Reuters/BBC |
| 中国政策/会议 | AnySearch（中文优先） |
| 国际财经/股市 | 子 Agent + Bloomberg/Reuters |
| 历史事实/制度 | 子 Agent + 维基百科/官方公告 |

### 核验输出格式（写入 doc_zst 的 verification 字段）

```json
"verification": "<事件/事实> ✓<来源>;<子明文章<日期>发布,事件<日期>确认=提前<N>天预测+剧本<完成度>"
```

参考真实兑现样例（claims.json 第 1 条）:

```
"verification": "2025-03-04长江和记宣布以228亿美元向BlackRock-TiL财团出售23国43个港口(含巴拿马)✓财新/BBC;中国外交部林剑3/5回应:对有关商业交易不予评论✓财新;香港特区政府2月斥责鲁比奥涉运河费说法'歪曲事实'✓BBC;子明文章2/7发布,事件3/4确认=提前25天预测+剧本100%走完"
```

## 6. 写入 doc_zst_xxx.json

```json
{
  "id": "doc_zst_<YYYY>_<topic>",
  "date": "<文章发布日期>",
  "title": "<文章原标题>",
  "summary": "<100-200字核心摘要>",
  "core_claim": "<一句话核心主张, fq=S 才填>",
  "snippet": "",
  "source_url": "<原文链接或 mp.weixin.qq.com/s/...>",
  "content": "# <标题>\n<清洗后正文>",
  "content_source": "corpus_md",
  "content_chars": <数字>,
  "fq": "A|S",
  "il": "|author_model|author_claim|source_text"
}
```

## 7. 兑现追踪（claims.json）

兑现后调用:

```json
{
  "id": "claim_<YYYY>_<seq>",
  "doc_id": "<对应 doc_zst_xxx>",
  "sw": <0-100 信号权重>,
  "claim": "<完整 claim 文本>",
  "verification": "<核验证据链>",
  "v": {
    "verdict": "fulfilled|partial|unfulfilled",
    "prediction": "<预测短摘>",
    "window_end": "<YYYY-MM-DD>",
    "verdict_date": "<YYYY-MM-DD>",
    "evidence": "<具体证据>",
    "sc": <0-100>,
    "sa": <0-100>
  }
}
```

sw/sc/sa 数值参考:
- sw < 50: 低信号预测
- sw 50-90: 重要预测
- sw 90-100: 神级预测（如李嘉诚案例 sw=98）

## 8. KPI 更新

每入库一篇 fq=S 文章，更新 overview.json:
- documents_total++
- documents_s++（仅 fq=S）
- documents_with_prediction++（仅 fq=S）
- 若新增 claim: claims_total++，并按信号级别增加 high_signal/top_signal

## 9. 试跑案例（武契奇文章，2026-07-13）

完整试跑结果见 thread 记录。要点:

- core_claim 提取成功: "武契奇辞职非退场而是'王车易位'..."
- 事实核验: WebSearch 返回空，**必须用子 Agent + BrightData**
- 核验发现: 武契奇**宣布**辞职（6/30）但**未真辞**（7/13 仍在任）
- 这恰好是兑现追踪的典型 case: 预测的"2 个月后选举"vs 实际"10-11 月选举"——可对比追踪
- 建议 fq: S（明确时序预测）
- 建议 il: author_model（"王车易位"框架）

## 10. 已知坑

1. **付费文无完整内容** — WeRSS 只抓到预览，跳过或标 "paywalled"
2. **HTML 格式** — 每条样式信息都要剥，可写脚本统一处理
3. **国际新闻** — WebSearch 经常空，必须子 Agent + 多源
4. **同一事件不同角度** — 多个 doc_zst 可能覆盖同一事件，需要用 doc_id 后缀区分（r/r2026/2026 模式）
5. **兑现时窗** — 无 window_end 的预测无法追踪，写 claims 时必须强制填
